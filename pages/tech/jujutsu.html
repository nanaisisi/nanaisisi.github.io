雑なJujutsuの使い方

概要

新しいGit互換型バージョン管理システム
その動作を一旦、Git/Githunへの適用を前提として大雑把に説明する。

Git/Githubへ適用する操作方法

自動コミットなのでjj addはなし。
併用型ローカルでは、
jj git init Githubなどの併用型リモートリポジトリでは、
jj git clone <URL> [<destination>]
jj commit -m "Update tutorial"
でやや違うらしい。
jj bookmark create doc-update -r @-
jj bookmark track doc-update
でブックマークする。
jj git push
でリモートリポジトリへ反映。

使わないやつ

jj init で新規jjリポジトリの初期化
jj describe -m "message" でコミットに近いが違うらしい。
